builder.dtd , builders as urls,people,images ...


<builder maintainer="mmbase.org" version="0">
This file contains all the settings for one builder

maintainer - maintainer of the builder defined as a domain
version - version of the builder



<status>active</status>
value - state of the builder either active or inactive



<classfile>Urls</classfile>
value - a optional tag, it names the classfile that allows you to add support routines or changes them on a per builder base. For example images has routines to image support.

possible values : 'Dummy' makes mmbase use the default setup, short name : 'Urls' will start class : org.mmbase.module.builders.Urls.class, long name : 'nl.vpro.mmbase.module.builders.Urls' will start that class


<searchage>31</searchage>
value - how old may the objects be when searching in a editor (this is a default value users can allways override).


<names>
names defines the different names used in user visible parts, singular for for talking about 1 object (Url,Person) and plurar for talking about multiple. all defined in

      <singular xml:lang="en">URL</singular>
      lang - Country code as defined by ISO 639
      value - Translation for that language

      <plural xml:lang="en">URLS</plural>
      lang - Country code as defined by ISO 639
      value - Translation for that language
</names>


<descriptions>
small description of the builder for human reading

      <description xml:lang="en">URL</description>
      lang - Country code as defined by ISO 639
      value - Translation for that language
</description>


<properties>
Define possible properties for this builder, to be used getInitParament(name)

      <property name="ImageConvert.ConvertorRoot" >/usr/local/</property>
      name - name of the property
      value - value of that property
</properties>



<fieldlist>
defines the different fields in this object, be carefull the order is important once defined keep them in the same order. If you use number and owner (the 2 default fields, please check the docs for this)
      <field>
           <gui>
                <guiname xml:lang="en">Owner</guiname>
                lang - by ISO 639
                value - Description of field in that language

                <guitype>integer</guiname>
                guitype - defines the gui type used in the editors, the basic ones are integer, string
                field,float,double,long but other types are possible and depend on your the editors you
                use (its guitypes).
           </gui>

           <editor>
                <positions>
                     <input>3</input>
                     value - the input/edit possition inside the editor (-1 if not shown)

                     <search>3</search>
                     value - the place of the field in the search selector (-1 if not shown)

                     <list>3</list>
                     value - the place of the field in the search result list (-1 if not shown)

                </positions>
           </editor>

           <db>
                <name>title</name>
                value - table name we want in our database (will be automapped if needed)

                <type state="persistent" notnull="true" key="true">STRING</type>
                state - storage type, persistant or virtual
                notnull - can this field be null, true of false
                key - is this field part of the builders key, true of false
                size - optional size of field (valid on STRING AND BYTE)
                value - MMBase storage type options : STRING, INTEGER, BYTE, DOUBLE,
                FLOAT,LONG
           </db>
      </field>
</fieldlist>

</builder>
